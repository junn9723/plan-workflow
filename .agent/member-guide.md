# メンバーガイド — 実行メンバー共通 (ビジネス版)

> 共通ルールは `CLAUDE.md`、ロール・ツール詳細は `.agent/config.yml` を参照。

---

## 1. メンバー行動規範

1. `.agent/tasks/` から自分のタスク(ブリーフ)を確認する
2. `.agent/knowledge.md` で関連知見を確認する
3. 既存の調査データ (`workspace/research/`) を確認し重複を避ける
4. エビデンスベースで作業する(主張には根拠を明記)
5. セルフレビューを実施する
6. タスクのステータスを更新する
7. 成果物は指定ディレクトリに保存する
8. 軽微な作業を除き、該当 Skill を積極的に使用する（一覧: `.agent/skills.md`）

---

## 2. ロール別作業手順

### リサーチャー (Researcher) — Phase 2

1. ブリーフの調査テーマ・範囲を確認
2. Web検索 (o3-search MCP) で情報収集
3. 必要に応じてPlaywright MCPでWebサイト閲覧・スクリーンショット取得
4. 収集データを構造化して `workspace/research/` に保存
5. 出典URL・レポート名・発行年を必ず明記
6. 複数ソースでクロスチェックし信頼性を評価
7. 調査サマリーを作成し、主要な発見事項をまとめる

**成果物**: `workspace/research/{テーマ名}.md` (出典付きの構造化データ)

### ストラテジスト (Strategist) — Phase 2-3

1. 調査データ (`workspace/research/`) を精読
2. 適切なフレームワーク(SWOT, 3C, 5Forces, BMC等)を選択・適用
3. 戦略オプションを整理し、推奨案を根拠とともに提示
4. 担当セクションを執筆 → `workspace/drafts/` に保存

**成果物**: `workspace/drafts/{セクション名}.md`

### アナリスト (Analyst) — Phase 2-3

1. 必要な市場データ・財務データを収集または受領
2. 定量モデルを構築(市場規模算定、収支計画、ユニットエコノミクス等)
3. 前提条件を全て明示
4. 可能な場合、楽観/標準/悲観の3シナリオを提示
5. 担当セクションを執筆 → `workspace/drafts/` に保存

**成果物**: `workspace/drafts/{セクション名}.md` (前提条件・算出ロジック付き)

### ライター (Writer) — Phase 3

1. 調査データと各セクションドラフトを精読
2. 全体の構成設計(目次・章立て)を策定
3. エグゼクティブサマリー・導入・結論等を執筆
4. 読者目線で分かりやすい表現・構成に整理
5. `workspace/drafts/` に保存

**成果物**: `workspace/drafts/{セクション名}.md`

### エディター (Editor) — Phase 4

**STEP A: 統合**
1. `workspace/drafts/` の全セクションを精読
2. 一つの統合ドキュメントに編集・統合
3. 数値・用語・表記の一貫性を確認
4. 論理の流れ・ストーリーの整合性を検証
5. 出典・エビデンスの漏れをチェック

**STEP B: レビュー判定**
- 問題なし → APPROVED → `docs/` に最終版配置
- 問題あり → 具体的な修正指示とともに担当者に差し戻し
- 差し戻しレコードを `workspace/reviews/` に保存

**成果物**: 統合済みドキュメント + レビューレコード

---

## 3. 重要ルール

- **全ての数値・主張にはエビデンス(出典)を明記する**
- **推定値には算出ロジックと前提条件を必ず付記する**
- **エディターによる統合レビューの省略は重大違反**
- **同一担当者で差し戻し3回 → PM差し戻し / 統合レビュー全体の上限は3回**
- 品質を左右する場面では深い思考で品質を高める
- 知見があれば `.agent/knowledge.md` に記録する
